1.Servlet简介

Server Applet

是用Java编写的服务器端程序。其主要功能在于交互式地浏览和修改数据，生成动态内容。

狭义的Servlet是指Java语言实现的一个接口

广义的Servlet是指任何实现了这个Servlet接口的类，一般情况下，人们将Servlet理解为后者。

Servlet运行于支持Java的应用服务器中。

从实现上讲，Servlet可以响应任何类型的请求，但绝大多数情况下Servlet只用来扩展基于HTTP协议的Web服务器。

故这些api中提供的一个接口叫做：servlet。若要开发servlet程序

  • 1.编写一个类，实现servlet接口
  • 2.把开发好的java类部署到web服务器中

ps: Servlet接口SUN公司定义了两个默认实现类，分别为：GenericServlet、HttpServlet

3.servlet原理/本质

浏览器发送一个HTTP请求，HTTP请求由Web容器(比如tomcat)分配给特定的Servlet进行处理
Servlet的本质是一个Java对象，这个对象拥有一系列的方法来处理HTTP请求。常见的方法有**doGet()，doPost()**等。

Web容器中包含了多个Servlet，特定的HTTP请求该由哪一个Servlet来处理是由Web容器中的web.xml来决定的。

Servlet工作流程分为三个阶段。init(初始化)，service(运行)，destroy(销毁)

Servlet没有main方法，所有行为由Container控制。Container就是一个java程序。

在加载Servlet的.class后，Servlet会由构造函数生成一个实例，然后Container调用init()方法完成参数的初始化，接着调用service()方法，service会根据网页的请求，调用doGet或者doPost方法(一般需要重写)，最后调用销毁方法。如图：

1.mapping问题

  • 一个servlet可以指定一个映射路径
<!--  注册servlet-->
<servlet>
    <servlet-name>hello</servlet-name>
    <servlet-class>com.heng.servlet.HelloServlet</servlet-class>
</servlet>
<!--1.一对一映射-->
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <url-pattern>/hello</url-pattern>
</servlet-mapping>
  • 一个servlet可以指定多个映射路径
<!--2 一对多映射-->
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <url-pattern>/hello</url-pattern>
</servlet-mapping>
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <url-pattern>/hello1</url-pattern>
</servlet-mapping>
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <url-pattern>/hello2</url-pattern>
</servlet-mapping>

当输入http://localhost:8080/s1/hello或 http://localhost:8080/s1/hello1或http://localhost:8080/s1/hello2

均可以访问

  • 一个servlet可以指定通用映射路径 
<!--3 通用映射-->
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <url-pattern>/hello/*</url-pattern>
</servlet-mapping>

当访问/hello/xxxx xxxx即* ==> 代指任何均可以访问

若改为<url-pattern>*</url-pattern>则报错
若改为 <url-pattern>/*</url-pattern> 则默认启动即相应的页面，index.jsp内容无法看到

  • 可以指定前缀或后缀

    <servlet-mapping>
  <servlet-name>hello</servlet-name>
  <url-pattern>*.heng</url-pattern>
</servlet-mapping>

    只要访问是以.heng为后缀则访问成功

2.优先级

  • 指定了固有的路径优先级最高，如果找不到找默认的

    示例：编写ErrorServlet 目的==> 访问错误到自己的页面，显示404

public class ErrorServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        resp.setCharacterEncoding("utf-8");
        resp.setContentType("text/html");
        PrintWriter writer = resp.getWriter();
        writer.print("<h1>404</h1>"); //页面显示404
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

4.ServletContext

web容器启动时，为每个web程序都创建了对应的ServletContext对象，代表当前的web应用

4.1可共享数据

( 在这个servlet保存的数据，可在另一个servlet中拿到)

HelloServlet02.java中

public class HelloServlet02 extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        ServletContext context = this.getServletContext();//servlet上下文
        String myName = "Yiheng";
        //将myName保存到ServletContext中
        context.setAttribute("Name",myName);
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

GetServlet.java中

public class GetServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        ServletContext context = this.getServletContext();
        //拿到HelloServlet02中保存的名字
        String usrName = (String) context.getAttribute("Name");
        resp.getWriter().print("Name " + usrName); //在浏览器中打印显示
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

web.xml文件

4.2可获取初始化参数

web.xml中 新加入参数

<context-param>
  <param-name>url</param-name>
  <param-value>jdbc:mysql://localhost:3306/mybatis</param-value>
</context-param>

创建GetParam.java

public class GetParam extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        ServletContext context = this.getServletContext();
        //通过ServletContext获得初始化的参数
        String url = context.getInitParameter("url");
        resp.getWriter().print(url);
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

web.xml中加入对应配置

<servlet>

    <servlet-class>com.heng.GetParam</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>getp</servlet-name>
    <url-pattern>/gp</url-pattern>
</servlet-mapping>

当访问http://localhost:8080/s2/gp 则数据已获取，即jdbc:mysql://localhost:3306/mybatis显示出来

3.可进行转发

新建Forward_demo.java 

public class Forward_demo extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        ServletContext context = this.getServletContext();
        //设置转发的请求路径
        RequestDispatcher requestDispatcher = context.getRequestDispatcher("/gp");
        //forward方法进行转发
        requestDispatcher.forward(req,resp);
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

web.xml 加入配置

<servlet>
    <servlet-name>forward_demo</servlet-name>
    <servlet-class>com.heng.Forward_demo</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>forward_demo</servlet-name>
    <url-pattern>/fd</url-pattern>
</servlet-mapping>

当访问http://localhost:8080/s2/fd时，此是会将/gp映射的内容转发过来，最终显示出来了jdbc:mysql://localhost:3306/mybatis

在GetResources.java中

public class GetResources extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        //找到xx.properties资源目录，输入为流
        InputStream stream = this.getServletContext().getResourceAsStream("/WEB-INF/classes/com/heng/xx.properties");
        //找db.properties
        //InputStream stream = this.getServletContext().getResourceAsStream("/WEB-INF/classes/db.properties");
        Properties prop = new Properties();
        prop.load(stream); //加载
        String name = prop.getProperty("username");
        String pwd = prop.getProperty("password");
        resp.getWriter().println("name : " + name);
        resp.getWriter().println("password : " + pwd);
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

web.xml中

<servlet>
    <servlet-name>resources</servlet-name>
    <servlet-class>com.heng.GetResources</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>resources</servlet-name>
    <url-pattern>/res</url-pattern>
</servlet-mapping>

当访问http://localhost:8080/s2/res则可看到对应资源被展示出来

5.Response重定向

新建RedirectServlet.java

public class RedirectServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        //重定向
        resp.sendRedirect("/s2/Name"); 
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

配置web.xml

<servlet>
  <servlet-name>redirect</servlet-name>
  <servlet-class>com.heng.RedirectServlet</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>redirect</servlet-name>
    <url-pattern>/rs</url-pattern>
  </servlet-mapping>

当输入http://localhost:8080/s2/rs 直接就跳转至http://localhost:8080/s2/Name进行访问

Forward是指内部转发。当一个Servlet处理请求的时候，它可以决定自己不继续处理，而是转发给另一个Servlet处理。

重定向是指当浏览器请求一个URL时，服务器返回一个重定向指令，告诉浏览器地址已经变了，麻烦使用新的URL再重新发送新请求。

相同点：页面均会跳转

不同：请求转发时，url不会发生变化(307)，而重定向(302)会

小案例 ==>

通过index.jsp上form表单输入账号，密码，进行提交，跳转到另一个页面，显示成功

需要在表单上指定action具体路径 ${pageContext.request.contextPath}指代当前项目

index.jsp

<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>Title</title>
</head>
<body>
<%--此时action找到当前项目，然后在当前项目找到login，进行请求--%>
<form action="${pageContext.request.contextPath}/login" method="get">
    用户名: <input type="text" name = "username"> <br>
    密码 : <input type="password" name="pwd"> <br>
    <input type="submit">
</form>
</body>
</html>

新建RequestServlet.java 进行处理，当点击提交之后重定向到相对页面success.jsp

public class RequestServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        String username = req.getParameter("username"); //获取账号密码
        String pwd = req.getParameter("pwd");
        System.out.println("账号:" + username); //控制台打印对应账号密码
        System.out.println("密码:" + pwd);
        resp.sendRedirect("/s2/success.jsp"); //重定向到对应页面，显示成功
    }
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

</body>
</html>
