java注解和反射

1.注解

1.1什么是注解

注解：Annotation(特殊的注释)

注释：comment

​ jdk5.0引入的新技术

作用：

  • 不是程序本身，可以对程序做出解释(与注释并无区别)
  • 可以被其他程序(如：编译器)读取

格式：

以 ”@注释名”在代码中存在，甚至可以添加一些参数

在哪使用:

可附加在 package，class，method，field上，相当于添加了额外的辅助信息

然后通过反射机制编程实现对这些元数据的访问

1.2内置注解

  • @Override：定义在java.lang.Override中，只适用于修饰方法
    • 表示一个方法声明打算重写父类的另一个方法声明
  • @Deprecated：定义在java.lang.Deprecated中，可修饰方法，属性，类
    • 表示不推荐使用此类元素，通常是因为危险或存在其余更好的选择
  • @SuppressWarnings：定义在java.lang.SuppressWarnings中，用于抑制编译时的警告信息
//需要添加参数才可以使用
@SuppressWarnings("all")
@SuppressWarnings("unchecked")

1.3元注解

(meta-annotation)

作用：解释其他注解, 提供对其他annotation类型的作说明

  • @Target：用于描述注解的使用范围(被描述的注解可以在哪用)
  • @Retention: 表示需要在什么级别保存该注解信息，用于描述注解的声明周期
    • SOURCE < CLASS < RUNTIME
  • @Document: 说明 注解将被包含在javadoc中
  • @Inherited: 说明子类可以继承父类中的该注解

1.4自定义注解

使用@interface自定义，自动继承了java.lang.annotation.Annotation接口

public class Main {
    //可以进行显示赋值,与顺序无关，若没有默认值，则必须给注解赋值
    @myAnnotation(age = 18,name = "yiheng")
    public void test() { }

    @myAnnotation01("Alice") // 此时可直接传入即可，不用写value = “Alice” (规范
    public void test01() {}
}

@Target({ElementType.TYPE,ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@interface myAnnotation{
    String name() default "";  //参数：  参数类型 + 参数名()
    int age() default 0;
    int id() default -1;

    String[] schools() default {"pku","cmu"};
}

@Target({ElementType.TYPE,ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@interface myAnnotation01{
    String value(); //若只要一个参数，用value
}

2.反射

Reflection

2.1java反射概述

反射机制允许程序在执行期借助于Reflection API取得任何类的内部信息，并能直接操作任意对象的内部属性及方法

Class c = Class.forName("java.lang.String")

优点 ：

  • 可以实现动态创建对象和编译
  • 灵活性高

缺点：

​ 对性能有影响，是一种解释操作，比直接执行相同的操作慢

2.6反射操作注解

package test.day05.test03;
import java.lang.annotation.*;
import java.lang.reflect.Field;

//反射操作注解
public class Main {
    public static void main(String[] args) throws ClassNotFoundException, NoSuchFieldException {
        Class c1 = Class.forName("test.day05.test03.Student");
        //反射获得注解
        Annotation[] annotations = c1.getAnnotations();
        for (Annotation annotation : annotations) {
            System.out.println(annotation);
        }
        //获得注解value的值
        TableHeng annotation = (TableHeng) c1.getAnnotation(TableHeng.class);
        String value = annotation.value();
        System.out.println(value);

        //获得类指定的注解
        Field name = c1.getDeclaredField("name");
        fieldHeng annotation1 = name.getAnnotation(fieldHeng.class);
        System.out.println(annotation1.columnName());
        System.out.println(annotation1.type());
        System.out.println(annotation1.length());
    }
}

@TableHeng("db_student")
class Student {
    @fieldHeng(columnName = "db_name",type = "int",length = 10)
    private String name;
    @fieldHeng(columnName = "db_age",type = "int",length = 10)
    int age;
    @fieldHeng(columnName = "db_id",type = "varchar",length = 3)
    int id;

    public Student(String name, int age, int id) {
        this.name = name;
        this.age = age;
        this.id = id;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                ", id=" + id +
                '}';
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public Student() {
    }
}


//类名的注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@interface TableHeng {
    String value();
}

//属性的注解
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@interface fieldHeng {
    String columnName();
    int length();
    String type();
}
