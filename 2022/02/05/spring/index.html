Spring

简介

Spring是一个轻量级的控制反转(IoC)**和面向切面(AOP)**的容器框架 (两个核心特性！！！)

Spring框架是由于软件开发的复杂性而创建的，简化开发

Spring使用的是基本的JavaBean来完成以前只可能由EJB完成的事情。然而，Spring的用途不仅仅限于服务器端的开发。从简单性、可测试性和松耦合性角度而言，绝大部分Java应用都可以从Spring中受益。

  • 创建 Spring 的主要目的是用来替代更加重量级的企业级 Java 技术

优点

  • 简化开发，解耦，集成其它框架。
  • 低侵入式设计，代码污染级别较低。
  • Spring的DI机制降低了业务对象替换的复杂性，提高了软件之间的解耦。
  • Spring AOP支持将一些通用的任务进行集中式的管理，例如：安全，事务，日志等，从而使代码能更好的复用。

Ioc思想

(Inversion of Control)

以前的思路

1.UserDao接口

public interface UserDao {
    void getUser(); //定义接口
}

2、UserDaoImpl实现类

public class UserDaoImpl implements UserDao{
    public void getUser() {
        System.out.println("获取用户数据");
    }
}

3、UserService业务接口

public interface UserService {
    void getUser();
}

4、UserServiceImpl业务接口

public class UserServiceImpl implements UserService{
    //引入Dao
    private UserDao userDao = new UserDaoImpl();

    public void getUser() {
        userDao.getUser();//业务层调用dao层
    }
}

Test类，用户测试

public class MyTest {
    public static void main(String[] args) {
        //实际调用业务层，dao层不接触
        UserService service = new UserServiceImpl();
        service.getUser();
    }
}

由于 private UserDao userDao = new UserDaoImpl();是程序主动创建对象

若又增加其他的内容，如UsermysqlImpl,UserSqlserverImpl....

此时，将需要进行对业务层UserServiceImpl.java进行修改代码(繁琐！！)

如何解决？！

将UserServiceImpl.java 中那条语句进行修改

private UserDao userDao;
//利用set动态实现值的注入
public void setUserDao(UserDao userDao) {
    this.userDao = userDao;
}

测试类MyTest.java进行调用

public class MyTest {
    public static void main(String[] args) {
        //实际调用业务层，dao层不接触
        UserService service = new UserServiceImpl();
        //调用UserServiceImpl中set
        ( (UserServiceImpl) service).setUserDao(new UserDaoImpl());
        service.getUser();
    }
}

若有mysql

//调mysql对应的
( (UserServiceImpl) service).setUserDao(new UserMysqlImpl()); 
//只需完成业务实现代码 即UserMysqlImpl.java的实现，  然后改变测试类即可

此时使用set注入后，程序不再具有主动性，而是被动的接收对象！

这种思想，使得程序员不需要管理对象的创建了，系统耦合性降低，只需要专注业务实现

这中思想即Ioc的原型

DI(依赖注入)是实现IoC的一种方法

Ioc（控制反转）即获得依赖的方式反转了！

总：

控制反转是一种通过描述（XML或注解）并通过第三方去生产或获取特定对象的方式。

在Spring中实现控制反转的是IoC容器，其实现方法是依赖注入（Dependency Injection,DI）。

AOP

(Aspect Oriented Programming) 面向切面编程

在运行时，动态地将代码切入到类的指定方法、指定位置上的编程思想就是面向切面的编程

旨在将**横切关注点与业务主体进行进一步分离，以提高程序代码的模块化程度。通过在现有代码基础上增加额外的通知**（Advice）机制，能够对被声明为“切点（Pointcut）”的代码块进行统一管理与装饰，如“对所有方法名以set*开头的方法添加后台日志”。

该思想使得开发人员能够将与代码核心业务逻辑关系不那么密切的功能（如日志功能）添加至程序中，同时又不降低业务代码的可读性。面向切面的程序设计思想也是面向切面软件开发的基础。

相当于我们的整个业务流程，被直接斩断，并在断掉的位置添加了一个额外的操作，再连接起来，也就是在一个切点位置插入内容。它的原理实际上就是通过动态代理机制实现的

Spring底层不是使用的JDK提供的动态代理，而是使用的第三方库实现，它能够以父类的形式代理，而不是接口

aop:pointcut 告诉spring，要切入的是哪个类/哪个方法

expression属性的execution填写格式如下：

修饰符 包名.类名.方法名称(方法参数)
  • 修饰符：public、protected、private、包括返回值类型、static等等（使用*代表任意修饰符）
  • 包名：如com.test（*代表全部，比如com.*代表com包下的全部包）
  • 类名：使用*也可以代表包下的所有类
  • 方法名称：可以使用*代表全部方法
  • 方法参数：填写对应的参数即可，比如(String, String)，也可以使用*来代表任意一个参数，使用..代表所有参数。

也可以使用其他属性来进行匹配，比如@annotation可以用于表示标记了哪些注解的方法被切入。

之后可添加执行前和执行后的动作 aop:after-returning 和 aop:before

我们想得到方法执行的一些参数，比如方法执行之后返回了什么，或是方法开始之前传入了什么参数等等。

在对应方法上加上JoinPoint point 

public void after(JoinPoint point) {
    System.out.println(Arrays.toString(point.getArgs()));//获得传入的参数，打印[你好]
    log.info("after....");
}

}

使用接口实现AOP

新建AopTest02.java

package com.heng.aop;

import org.springframework.aop.MethodBeforeAdvice;

import java.lang.reflect.Method;
import java.util.Arrays;

//此处也可以继承AfterReturningAdvice 来实现方法之后的操作
public class AopTest02 implements MethodBeforeAdvice {

    @Override
    public void before(Method method, Object[] objects, Object o) throws Throwable {
        System.out.println("方法名字" + method.getName());
        System.out.println("方法参数" + Arrays.toString(objects)); 
        System.out.println("方法执行的对象" + o);
        System.out.println("通过接口实现aop");
    }
}

进行对应配置

<bean id="aopTest02" class="com.heng.aop.AopTest02"></bean>
<aop:config>
    <aop:pointcut id="stu4" expression="execution(* com.heng.entity.Student.say(String))"/>
    <aop:advisor advice-ref="aopTest02" pointcut-ref="stu4"></aop:advisor>
</aop:config>

一些术语

  • 通知（Advice）: AOP 框架中的增强处理，通知描述了切面何时执行以及如何执行增强处理，也就是我们上面编写的方法实现。
  • 连接点（join point）: 连接点表示应用执行过程中能够插入切面的一个点，这个点可以是方法的调用、异常的抛出，实际上就是我们在方法执行前或是执行后需要做的内容。
  • 切点（PointCut）: 可以插入增强处理的连接点，可以是方法执行之前也可以方法执行之后，还可以是抛出异常之类的。
  • 切面（Aspect）: 切面是通知和切点的结合，我们之前在xml中定义的就是切面，包括很多信息。
  • 引入（Introduction）：引入允许我们向现有的类添加新的方法或者属性。
  • 织入（Weaving）: 将增强处理添加到目标对象中，并创建一个被增强的对象，我们之前都是在将我们的增强处理添加到目标对象，也就是织入

利用bean注解

在类中的方法中 只需要编写一个方法，并返回我们要创建的Bean的对象即可，并在其上方添加@Bean注解

比如先创建Student.java 类

package com.heng.bean;

import lombok.ToString;

@ToString
public class Student {
    String name;
    int age;

    public void setName(String name) {
        this.name = name;
    }
}

只需在刚刚的MainConfig.java 中添加

@Bean
// @Scope("prototype") 可以指定范围
public Student student() {
    Student student = new Student();
    student.setName("yiheng"); //设置对应的名字
    return student; //进行返回
}

在主类Main.java中进行测试

package com.heng.test;

import com.heng.Config.MainConfig;
import com.heng.bean.Student;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class Main {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext context = new AnnotationConfigApplicationContext(MainConfig.class);
        Student student = context.getBean(Student.class);
        System.out.println(student);
    }
}

运行打印出 Student(name=yiheng, age=0) 同时可观察具体信息

利用Component注解

直接在类上添加@Component注解来将一个类进行注册，但要添加相关扫描

来告诉Spring需要在哪些包中查找我们提供@Component声明的Bean

即在配置类上添加一个@ComponentScan注解

此时配置类变为

@Configuration

public class MainConfig {   }

对应的Student类上加上@Component

package com.heng.bean;

import lombok.ToString;
import org.springframework.stereotype.Component;

@ToString
@Component
public class Student {

    int age;

    }
}

package com.heng.test;

自动装配

  • @Resource注解

  • @Autowired 注解（IDEA不推荐将其使用在字段上，会出现黄标，但是可以放在方法或是构造方法上）

    • @Resource默认ByName如果找不到则ByType，可以添加到set方法、字段上。
    • @Autowired默认是byType，可以添加在构造方法、set方法、字段、方法参数上。

还可以通过@PostConstruct注解来添加构造后执行的方法，它等价于之前讲解的init-method

@PostConstruct
public void init(){
    System.out.println("我是初始化方法！");
}

执行顺序：Constructor(构造方法) –> @Autowired(依赖注入) –> @PostConstruct

同时也要对应销毁方法@PreDestory注解

  • 如果要注册为Bean的类是由其他框架提供，我们无法修改其源代码，那么我们就使用第一种方式进行配置。
  • 如果要注册为Bean的类是我们自己编写的，我们就可以直接在类上添加注解，并在配置中添加扫描。

@Aspect

package com.heng.Config;

import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.ComponentScans;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.EnableAspectJAutoProxy;

@Configuration
@ComponentScans( {
        @ComponentScan("com.heng.bean"),
        @ComponentScan("com.heng.aop")
})
@EnableAspectJAutoProxy
public class MainConfig {}

在Student.java 中添加对应的方法，比如say表明要在此方法前/后切入对应的操作

@Deprecated 
public String say(String msg) {
    System.out.println("我是say...");
    return msg;
}

在aop类中，添加before和 after逻辑

package com.heng.aop;

import lombok.extern.java.Log;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.springframework.stereotype.Component;

@Component
@Aspect 
@Log //利用log打印一些信息
public class AopTest {
    @Before("execution(* com.heng.bean.Student.say(String))")
    public void before() {
        log.info("我是before要在say之前执行....");
    }
    @After("@annotation(Deprecated)") //因为say方法前添加了对应注解，故可以利用此方法
    public void after() {
        log.info("我是after，在say之后切入执行");
    }
}
